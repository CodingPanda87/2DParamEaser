# ParamEaser — 设计文档

> 从 DXF 图纸提取尺寸标注，绑定参数控件，生成带交互标识的 SVG 预览文件。
> 版本 1.0 — 2026-06-14

---

## 1. 项目概述

### 1.1 目标

机械工程师在处理 DXF 图纸时，需要将图纸中的尺寸标注与参数化控件关联起来。ParamEaser 提供一套桌面 Web 应用工作流：

1. 加载 DXF 图纸并解析其中的尺寸标注实体
2. 在图纸上以标记点形式显示所有尺寸的中心坐标
3. 允许用户为每个尺寸绑定参数控件类型（float/int）和参数 key
4. 将图纸、尺寸线、参数控件标识合并输出为 SVG 文件
5. 在浏览器中预览 SVG，查看已绑定的参数控件

### 1.2 目标用户

机械工程师，用于处理机械加工图纸、零部件图纸中的尺寸参数化。

### 1.3 技术平台

桌面 Web 应用，在浏览器中运行。

## 2. 交互界面布局

### 2.1 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  ParamEaser  |  drawing-001.dxf      [打开] [导出SVG]     │
├────────────────────────┬────────────────────────────────┤
│                        │                                │
│  原始 DXF 图纸          │  SVG 输出预览                   │
│  (可交互标记点)          │  (纯展示，含参数控件标签)         │
│                        │                                │
│  🟠 D1  (未绑定)        │  ┌──────────────────┐         │
│  🟢 radius (已绑定)     │  │  $width: float    │         │
│                        │  │  $radius: float   │         │
│  ↔ 可拖动分割线          │  └──────────────────┘         │
│                        │                                │
├────────────────────────┴────────────────────────────────┤
│  文件信息 | 尺寸: 12个 | 已绑定: 3/12 | ⚠ 未绑定: 9      │
└─────────────────────────────────────────────────────────┘
```

### 2.2 布局分区

| 区域 | 内容 | 交互性 |
|------|------|--------|
| **左面板** | 原始 DXF 渲染 + 尺寸中心坐标标记点 | 可交互 — 点击标记弹窗编辑 |
| **右面板** | SVG 输出预览（图纸 + 尺寸线 + 控件标签） | 只读预览 |
| **分割线** | 左右面板中间的可拖动分割条 | 可拖动调整面板宽度 |
| **顶部栏** | 应用名称 + 当前文件名 + 操作按钮 | 打开文件 / 导出SVG |
| **状态栏** | 文件信息、尺寸统计、绑定进度 | 只读 |

### 2.3 尺寸标记点设计

- 每个尺寸的中心坐标处以圆形标记点显示
- **🟠 橙色标记 + Dn 编号** = 未绑定参数
- **🟢 绿色标记 + 参数 key 名** = 已绑定参数
- 点击任意标记点 → 弹出 Popover 编辑卡片
- 标记点图形始终保持在可见大小，不随图纸缩放而消失

### 2.4 Popover 编辑卡片

点击标记点后，在标记点附近弹出编辑卡片，包含：

| 字段 | 说明 |
|------|------|
| 尺寸信息 | ID、类型、值、中心坐标（只读，用于确认） |
| **参数类型** | float / int 切换按钮 |
| **参数 Key** | 文本输入框，支持前缀 `$` 提示 |
| 默认值 | 可选，预填当前尺寸值 |
| 操作按钮 | [✓ 绑定] [取消] |

## 3. 尺寸标注类型与视觉样式

### 3.1 支持的四种类型

| 类型 | DXF 标识 | 说明 | 控件标签方向 |
|------|---------|------|------------|
| **对齐标注 (Aligned)** | `DIMENSION` type=0 | 斜边实际长度，尺寸线与测量方向平行 | 沿尺寸线旋转 |
| **线性标注 (Linear)** | `DIMENSION` type=1 (水平/垂直) | 水平或垂直投影距离 | 水平/垂直 |
| **半径/直径标注 (Radial/Diameter)** | `DIMENSION` type=4/3 | 圆或圆弧的半径/直径 | 水平，靠近标注线末端 |
| **角度标注 (Angular)** | `DIMENSION` type=5 | 两条线之间的角度 | 沿弧线方向或水平 |

### 3.2 SVG 控件标签规则

- 位置：替换或紧邻原始尺寸文字位置（即尺寸标注的中心坐标点）
- 方向：与对应尺寸线平行（对齐标注沿斜线旋转，线性标注按方向水平/垂直）
- 格式：`` $key: type ``（等宽字体）

### 3.3 参数控件视觉编码

| 状态 | 颜色 | 样式 | 示例 |
|------|------|------|------|
| **float 已绑定** | 🔵 蓝色 `#4fc3f7` | 实线标签 | `$hole_dia: float` |
| **int 已绑定** | 🟢 绿色 `#4caf50` | 实线标签 | `$teeth: int` |
| **带默认值** | 🟪 紫色 `#ce93d8` | 标签 + 数值 | `$angle: float | 45.0` |
| **未绑定** | 🟠 橙色 `#ff9800` | 虚线标签 | `D5 ??? unbind` |

## 5. 存储结构

### 5.1 项目文件 (.parameaser)

JSON 格式，与 DXF 文件同名存储。

```json
{
  "version": "1.0",
  "source": "drawing-001.dxf",
  "dimensions": [
    {
      "id": "D1",
      "type": "aligned",
      "value": 120.5,
      "unit": "mm",
      "center": { "x": 48.0, "y": 68.0 },
      "rotation": -56.0,
      "param": {
        "key": "slant_length",
        "type": "float",
        "default": 120.5
      }
    },
    {
      "id": "D2",
      "type": "linear",
      "orientation": "horizontal",
      "value": 200.0,
      "unit": "mm",
      "center": { "x": 150.0, "y": 22.0 },
      "rotation": 0.0,
      "param": null
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 自动编号 D1, D2, D3… |
| `type` | string | `aligned` / `linear` / `radial` / `diameter` / `angular` |
| `orientation` | string | 仅 linear: `horizontal` / `vertical` |
| `value` | number | 尺寸标注的数值 |
| `unit` | string | 单位（从 DXF 读取或默认 mm） |
| `center` | object | 尺寸文字的中心坐标 `{x, y}` |
| `rotation` | number | 尺寸线旋转角度（度，用于控件标签对齐） |
| `param` | object/null | `null` = 未绑定；绑定后包含 `key`, `type`, `default` |

### 5.2 SVG 输出文件

SVG 按图层组织：

```
<svg>
  <g id="drawing">        — 2D 图纸实体（从 DXF 转换）
  <g id="dimensions">     — 尺寸标注线 + 箭头 + 数值
  <g id="params">         — 参数控件标签（已绑定/未绑定）
  <metadata>              — JSON 元数据（与 .parameaser 一致）
</svg>
```

控件标签在 SVG 中使用 `<rect>` + `<text>` 实现，绑定 CSS 类名 `param-float` / `param-int` / `param-unbound`。

### 5.3 文件组织

每个图纸对应一个目录：

```
drawing-001/
├── drawing-001.dxf           — 源文件
├── drawing-001.parameaser    — 项目文件 (JSON)
├── drawing-001.svg           — 输出 SVG
└── preview.html              — 预览页 (可选)
```

## 7. 功能范围与边界

### 7.1 范围之内 (MVP)

- 从 DXF R12/R2000 格式中解析 DIMENSION 实体
- 支持四种标注类型：对齐、线性、半径/直径、角度
- 在左侧 DXF 渲染图上显示尺寸中心标记点
- 点击标记点弹出 Popover 绑定 float/int + key
- 生成包含参数控件标签的 SVG 文件
- 浏览器预览 SVG

### 7.2 范围之外 (后续版本)

- DXF 写入/回存
- 参数值联动计算
- 批量处理多文件
- 导出参数列表（JSON/CSV）
- 3D 视图支持
- 多用户协作
- 历史版本对比

## 8. 设计决策记录

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 参数编辑位置 | 右侧面板 / 浮动弹窗 / 底部面板 | **左面板Popover** | 直接在标记点处编辑，操作路径最短 |
| SVG 控件风格 | 标签内嵌 / 外部标注 | **标签内嵌** | SVG 自包含，不依赖外部映射表 |
| 项目文件格式 | JSON / YAML / SQLite | **JSON** | 简单可靠，可读性强，易版本控制 |
| 布局结构 | 左图右属性 / 上图下列表 | **左原始图+右预览** | 同时看到原始图纸和最终效果 |
| 控件颜色编码 | 统一色 / 按类型分色 | **float蓝/int绿** | 一眼区分参数类型 |

## 9. 技术考量

### 9.1 DXF 解析

- 优先支持 DXF R12 ASCII 格式（最广泛兼容）
- 需要解析的 DXF 组码：
  - `(0, "DIMENSION")` — 实体类型
  - `(100, "AcDbAlignedDimension")` / `(100, "AcDbRotatedDimension")` — 子类标记
  - `(70)` — 标注类型标志位
  - `(10, 20)` — 定义点
  - `(11, 21)` — 文字中点
  - `(13, 23)` / `(14, 24)` — 第一/第二延伸线起点
  - `(40)` — 标注值
  - `(50)` — 旋转角度
  - `(1)` / `(42)` — 标注文字内容/实际测量值

### 9.2 SVG 生成

- 使用纯 SVG DOM 操作，无 Canvas 依赖
- 三个 `<g>` 图层：drawing / dimensions / params
- 支持 viewBox 自适应缩放
- 控件标签位置基于尺寸中心坐标计算，旋转角度跟随尺寸线方向
- SVG 内嵌 `<style>` 定义控件样式类
